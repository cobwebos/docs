# <a name="glossary"></a>术语表

## <a name="a"></a>A

### <a name="acoustic-model"></a>声学模型

声学模型是给定语言的分类器，用于将短小声音片段标记并归入许多音素或音位之一。 例如，“speech”一词由四个音素“s p iy ch”组成。 这些分类大约每秒执行 100 次

## <a name="b"></a>B

## <a name="c"></a>C

### <a name="conversational-model"></a>对话模型

适用于识别对话形式语音的模型。 Microsoft 对话 AM 通常适用于处理对另一个人说出的语音。

## <a name="d"></a>D

### <a name="deployment"></a>部署

调整后的自定义模型成为服务并公开 URI 的过程

## <a name="e"></a>E

## <a name="f"></a>F

## <a name="g"></a>G

## <a name="h"></a>H

## <a name="i"></a>I

### <a name="inverse-text-normalization"></a>反向文本规范化

将“原始”的无格式文本转换回格式化文本的过程（即区分大小写且有标点），称为“反向文本规范化 (ITN)”。

## <a name="j"></a>J

## <a name="k"></a>K

## <a name="l"></a>L

### <a name="language-model"></a>语言模型

语言模型是字词序列的概率分布。 语言模型有助于系统根据字词序列本身的概率，在发音类似的字词序列中做出决定

## <a name="m"></a>M

## <a name="n"></a>N

### <a name="normalization"></a>规范化

规范化（文本）：将生成的文本（即听录）转换为系统可读的标准明确形式。

## <a name="o"></a>O

## <a name="p"></a>P

## <a name="q"></a>Q

## <a name="r"></a>R

## <a name="s"></a>S

### <a name="search-and-dictate-model"></a>搜索和听写模型

适用于处理命令的声学模型。 Microsoft 搜索和听写 AM 适用于处理对应用或设备说出的语音（如命令）

### <a name="subscription-key"></a>订阅密钥

订阅密钥是字符串，必须指定为查询字符串参数，才能调用任何自定义语音服务模型。 订阅密钥是从 [Azure 门户](https://portal.azure.com/#create/Microsoft.CognitiveServices/apitype/CustomSpeech)获得。获得后，便可以在自定义语音服务门户的“我的订阅”中找到。

## <a name="t"></a>T

### <a name="transcription"></a>听录

听录：处理一段音频 .wav 文件所生成的文本片段
