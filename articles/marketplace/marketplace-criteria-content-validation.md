# <a name="azure-marketplace-listing-guidelines"></a>Azure 市场列表准则

本文介绍了在 Azure 市场上列出新产品和服务的要求和准则。 所有产品/服务必须满足 "[列出所有产品/服务的要求](#listing-requirements-for-all-offers)" 部分中列出的要求。 使用右侧的链接跳转到特定列表类型的要求和清单。

## <a name="listing-requirements-for-all-offers"></a>关于列出所有产品/服务的要求

| 不是。 | 列表元素 | 基本要求 | 最佳要求 |
|:--- |:--- |:--- |:--- |
| 1 | 潜在客户目标 | 已配置潜在顾客目标。| One Commercial Partner (OCP) 目录具有合作伙伴解决方案选项卡中也列有的潜在客户目标 CRM 信息。 |
| 2 | 套餐标题 | 简要介绍解决方案产品/服务。 与合作伙伴网站上解决方案的在线推广信息相匹配。 | 包含搜索关键字。 |
| 3 | 徽标 | 徽标显示正确。 | 徽标显示正确。 |
| 4 |  套餐说明 | <ul><li> 包含 2 - 3 段。</li><li>能够快速轻松地了解解决方案产品/服务。</li><li>不存在拼写和语法错误。</li><li>内容全面，可捕获目标受众、用户类型及其价值所在（价值主张）。</li><li>采用叙述性的段落，包含易于理解的短句。</li></ul> | <ul><li> 概述目标行业（如果相关）。</li><li>每个段落标题采用适当的格式，后接一句话或短语式的汇总内容，并通过概括性的要点（如果适用）强调重要优势。 其目标是使读者无需阅读长段就能通过格式快速轻松地了解产品/服务。</li><li>每个段落之间都有间距。 它看起来像汽车手册。 也就是说，它全面且概要性地描述产品/服务，不含任何技术术语。 | 
| 5 | 类别和行业 | <ul><li>类别和子类别必须与产品/服务匹配 </li><li>不要选择无法满足您的产品/服务功能的类别/子类别。 </li></ul> | <ul><li>最多选择两个类别，包括主类别和辅助类别 (可选) 。</li><li>对于每个主类别和/或辅助类别，最多选择两个子类别。 如果未选择子类别，你的产品/服务将仍然可在所选类别上发现。</li></ul> |
| 6 | 映像 | <ul><li>合作伙伴中心列出了映像要求。</li><li>屏幕截图中包含的文本清晰易读，包含的映像清晰。 | 能够快速轻松地了解解决方案产品/服务。 |
| 7 | 视频 | <ul><li>不需要视频，但如果提供，视频在播放时不能出现任何错误。</li><li>如果提供视频，除非要演示迁移解决方案，否则不能提到竞争对手公司。 |<ul><li>理想情况下，长度为 3 分钟或更长。</li><li>通过视频内容能够轻松了解解决方案产品/服务。</li><li>提供解决方案功能演示。 |
| 8 | 列表状态（行动号召） | <ul><li>必须标记为以下类型之一： <ul><li>*与我联系*</li><li>试用/立即试用/开始试用/测试驱动器   </li><li>立即购买/立即获取 </li></ul></ul> | 客户能够马上了解后续步骤： <ol><li>试用。</li><li>立即购买。</li><li>通过电子邮件或电话号码取得联系，以安排概念证明 (POC)、评估或简报。</li></ol> | 
| 9 | 解决方案定价 | 必须提供解决方案定价选项卡/详细信息，并且定价必须采用合作伙伴解决方案产品/服务的本地货币来表示。 | 分层定价应提供多种计费选项以供客户选择。 |
| 10 | 了解详细信息 | 底部的链接（在说明下面，不是左侧的 Azure 市场链接）指向有关解决方案的详细信息，并正确公开显示。 |    链接到特定的项（例如合作伙伴站点上的规格页），而不只是链接到合作伙伴主页。 | 
| 11 | 解决方案支持和帮助 | 链接到下面至少一项： <ul><li>电话号码</li><li>电子邮件支持</li><li>聊天代理</li><li>社区论坛 |<ul><li>列出所有支持方法。</li><li>在试用或体验期免费提供付费支持 。 |
| 12 | 合法 | 通过公共 URL 提供策略或条款。 |  |

|:--- |:--- |:--- |:--- |
| 1 | 套餐标题 |<ul><li>只能包含小写的字母、字母数字字符、短划线或下划线。 标题在发布后无法修改。</li><li>描述解决方案产品/服务。</li><li>匹配合作伙伴网站中的解决方案在线推广。 | 包含搜索关键字。 |
| 2 | 技术信息：配置 |<ul><li>对于软件即服务 (SaaS) 应用，请选择是仅列出应用，还是要让客户通过 Azure 购买应用。</li><li>选择要在产品/服务购买按钮上显示的文本：“免费”、“免费试用”或“与我联系”  。</li><li>在弹出框口中，如果应用利用该技术，请仅选择一个适用产品：Cortana Intelligence、Power BI 解决方案模板或 Power Apps。 |  |
| 3 | 体验版 | 选择“是”或“否”  | 客户能够马上了解后续步骤： <ol><li>试用。</li><li>立即购买。</li><li>通过电子邮件或电话号码取得联系，以安排概念证明、评估或简报。</li></ol> |
| 4 | 店面详细信息：套餐摘要 | 显示在应用的搜索页上，最多包含 100 个字符。 | |
| 5 | 店面详细信息：行业 | 行业（最多 2 个）：选择应用最适合的行业。 | |
| 6 | 套餐说明 | <ul><li>允许使用简单的 HTML，包括、、、、 \<p> \<em> \<ul> \<li> \<ol> 和标头标记。 最大 3000 个字符。</li><li>包含 2 - 3 段。</li><li>能够快速轻松地了解解决方案产品/服务。</li><li>说明内容全面，可捕获目标受众、用户类型及其价值所在（价值主张）。</li><li>采用叙述性的段落，包含易于理解的短句。 | <ul><li>概述目标行业（如果相关）。</li><li>每个段落标题采用适当的格式，后接一句话或短语式的汇总内容，并通过概括性的要点（如果适用）强调重要优势。 其目标是使读者无需阅读整个段落就能通过格式快速轻松地了解产品/服务。</li><li>每个段落之间都有间距。 它看起来像汽车手册。 也就是说，它全面且概要性地描述产品/服务，不含任何技术术语。 | 
| 7 | 营销项目 | 徽标正确显示。 | <ul><li>徽标： Small (48 x 48 像素) ，中 (90 x 90 像素) ，216 x 216 和 350 x 350 像素之间的大型 (是必需的。</li><li>最大屏幕截图 (。5) ：需要。分辨率为1280x720 像素的 PNG 图像。 |
| 8 | 类别和行业 | <ul><li>类别与解决方案套餐的功能相匹配。</li><li>在弹出窗口中选择至少一项。</li><li>不要选择解决方案不适合的类别。</li><li>最佳行业，或选定的所有行业（如果不是最佳行业）。 | 最大 选择 3 个类别（如果适用）。 |
| 9 | 潜在顾客管理 | 选择用于存储潜在顾客的系统。 请参阅[获取潜在顾客](./partner-center-portal/commercial-marketplace-get-customer-leads.md)以连接 CRM 系统。 |  |
| 10 | 联系人：解决方案支持和帮助 | <ul><li>工程联系人姓名：应用的工程联系人姓名。 此联系人将收到来自 Microsoft 的技术邮件。</li><li>工程联系人电子邮件：应用的工程联系人的电子邮件地址。</li><li>工程联系人电话：工程联系人的电话号码。 支持 [ISO 电话号码表示法](https://en.wikipedia.org/wiki/E.123)。</li><li>支持联系人姓名：应用的支持联系人姓名。 此联系人将收到来自 Microsoft 的支持相关邮件。</li><li>支持联系人电子邮件：应用的支持联系人的电子邮件地址。</li><li>支持联系人电话：支持联系人的电话号码。 支持 [ISO 电话号码表示法](https://en.wikipedia.org/wiki/E.123)。</li><li>支持 URL：支持页的 URL。 | <ul><li>列出所有支持方法。</li><li>在试用或体验期免费提供付费支持 。 |
| 11 | 合法 |<ul><li>隐私策略 URL：CPP 的“隐私政策 URL”字段中应用隐私策略的 URL。</li><li>使用条款：应用的使用条款。 客户必须接受这些条款才能试用应用。 |    通过公共 URL 站点提供政策或条款。 |

| 9 | 资质 | 至少必须具备以下其中一项资格：<ul><li>应用程序开发</li><li>应用程序集成</li><li>应用程序生命周期管理</li><li>云平台<li>数据分析</li><li>数据中心</li><li>数据平台</li><li>DevOps |
