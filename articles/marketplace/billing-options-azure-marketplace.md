# <a name="billing-options-in-the-azure-marketplace"></a>Azure 市场的计费选项

本文介绍 [Azure 市场](https://azuremarketplace.microsoft.com)中提供的计费选项。

## <a name="commercial-considerations-in-the-marketplace"></a>市场的商业注意事项
市场不会分享以下列表类型的收入： 
*   列出
*   试用
*   交易：使用自带许可 (BYOL) 计费模型

你不会因加入市场中的店面而被收取额外费用。

有关详细信息，请参阅 [Microsoft Azure 市场参与策略](https://azure.microsoft.com/support/legal/marketplace/participation-policies)。  

## <a name="pay-as-you-go-and-byol-billing-options"></a>即用即付和 BYOL 计费选项
使用“即用即付”计费模型作为发布选项时，基于使用情况的软件许可收入将在你与 Microsoft 之间以 80%/20% 的比例分享。 你可以使用“即用即付”和 BYOL 计费模式为单个产品/服务定价。 这两种计费模型可作为单独的 SKU 在产品级别共存。 可以在云合作伙伴门户的产品/服务中配置计费模型。 

请考虑以下示例：
*   如果启用“即用即付”选项，则会得到以下结果：
    | 你的许可证费用 | 每小时 $1.00 |
    |:--- |:--- |
    | Azure 使用费用（D1/1 核） | 每小时 $0.14 |
    | *由 Microsoft 向客户收费* | *每小时 $1.14* |

    在此场景中，Microsoft 按每小时 $1.14 收取所发布的 VM 映像的使用费用：
    | Microsoft 收费 | 每小时 $1.14 |
    |:--- |:--- |
    | Microsoft 将许可证费用的 80% 支付给你 | 每小时 $0.80 |
    | Microsoft 保留许可证费用的 20% | 每小时 $0.20 |
    | Microsoft 保留 Azure 使用费用 | 每小时 $0.14 |

*   如果启用“BYOL”选项，则会得到以下结果：
    | 你的许可证费用 | 许可证费用由你协商确定和收取 |
    |:--- |:--- |
    | Azure 使用费用（D1/1 核） | 每小时 $0.14 |
    | *由 Microsoft 向客户收费* | *每小时 $0.14* |

    在此场景中，Microsoft 按每小时 $0.14 收取所发布的 VM 映像的使用费用： 
    | Microsoft 收费 | 每小时 $0.14 |
    |:--- |:--- |
    | Microsoft 保留 Azure 使用费用 | 每小时 $0.14 |
    | Microsoft 保留许可证费用的 0% | 每小时 $0.00 |

## <a name="single-billing-and-payment-methods"></a>单独计费和付款方式
使用市场中的“交易”发布选项的一个重要好处是，你的许可成本和 Azure 使用情况是直接向你的客户单独收取的。

在这种情况下，Microsoft 会以你的名义开账单并收集。 Microsoft 计费不需要你与你的客户建立自己的采购关系。 单独计费可以节省你的时间和资源。 它还可以帮助你专注于经营销售而不是收取账款。 

## <a name="enterprise-agreement"></a>企业协议  
如果你是 Microsoft 企业协议客户，则可以使用企业协议来支付 Microsoft 产品。 你可能会因使用产品（包括因 Azure 使用情况）而被收取费用。 使用企业协议进行支付适用于希望三年或更长时间获得软件和云服务许可的组织。 你可以分期付款，而非预先一次 付清。 如果使用“即用即付”发布选项，则软件许可成本计费将采用按季度计算的企业协议超额计费周期。  

### <a name="monetary-commitment"></a>货币承诺
如果你是企业协议客户，可以将 Azure 添加到你的协议。 通过向 Azure 做出前期货币承诺，将 Azure 添加到协议中。 货币承诺在一整年内有效。 你的承诺包括使用 Azure 服务的任何组合。
