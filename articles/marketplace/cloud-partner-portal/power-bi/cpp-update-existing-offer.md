# <a name="update-a-power-bi-app-offer"></a>更新 Power BI 应用服务

本文介绍如何在[云合作伙伴门户](https://cloudpartner.azure.com/)中更新 Microsoft Power BI 应用产品/服务，然后重新发布该产品/服务。 以下是更新产品/服务的一些常见原因：

- 若要更新中的应用内容 Power BI 并从新打包的应用获取安装 URL
- 更新产品/服务的 Azure Marketplace 元数据（销售、营销或支持信息和资产）
 
使用门户的 "**比较**和**历史记录**" 功能跟踪所做的更改。

## <a name="common-update-operations"></a>常见更新操作

在 AppSource 中发布产品/服务后，不能更改 Power BI 应用服务的某些属性。 不允许的更改包括**产品 id**和**发布者 id**。 但你仍可以更改各种特征。 下面是一些常见更改。

### <a name="update-app-content-in-power-bi"></a>更新 Power BI 中的应用内容

通常会定期更新 Power BI 应用，其中包含新内容、安全修补程序、功能等。 当应用程序更改时，请按照以下步骤更新其安装 URL：

1.  登录[云合作伙伴平台](https://cloudpartner.azure.com/)。
2.  在“所有套餐”下，找到要更新的套餐。
3.  在 "**技术信息**" 选项卡上，输入新的安装 URL。
4.  选择 "**发布**" 以启动工作流，将新的应用版本发布到 AppSource。


### <a name="update-the-offers-marketplace-metadata"></a>更新产品/服务的 Marketplace 元数据

产品/服务的 Marketplace 元数据包含公司名称、徽标和其他信息。 若要更新元数据，请执行以下步骤：

1.  登录[云合作伙伴平台](https://cloudpartner.azure.com/)。
2.  在“所有套餐”下，找到要更新的套餐。
3.  在 "**店面详细信息**" 选项卡上，按照[Power BI 应用店面详细信息 "选项卡](./cpp-storefront-details-tab.md)一文中的说明更改元数据。
4.  选择“发布”，启动发布更改的工作流。


## <a name="the-compare-feature"></a>比较功能

更新已发布的产品/服务时，可以检查所做的更改。 使用 "**比较**" 功能：

1.  在编辑过程中的任意位置，选择产品/服务的 "**比较**" 按钮。

    ![比较按钮](./media/compare-feature-button.png)

2.  并列查看营销资产和元数据的版本。


## <a name="history-of-publishing-actions"></a>发布操作的历史记录

若要查看产品/服务的发布历史记录，请在云合作伙伴门户左侧打开 "**历史记录**" 选项卡。在这里，你将看到 AppSource 提供的时间戳操作的历史记录。
