# <a name="power-bi-app-offer"></a>Power BI 应用套餐

|              |                                |
|--------------|--------------------------------|
| 本文介绍如何将 Power BI 应用程序发布到 Microsoft [AppSource marketplace](https://appsource.microsoft.com/)。  Power BI 应用包 Power BI 内容（包括数据集、报表和仪表板）可自定义。 然后，你可以通过 AppSource 将应用部署到其他 Power BI 租户，执行开发人员允许的调整和自定义，并将其连接到你自己的数据。 | ![Power BI 图标](./media/powerbi-icon.png) |


本文分为三个主要部分：

-   [先决条件](./cpp-prerequisites.md)。 创建和发布 Power BI 应用程序产品/服务的技术和业务要求。
-   [创建 Power BI 应用服务](./cpp-create-offer.md)。 如何使用[云合作伙伴门户](https://cloudpartner.azure.com)创建 Power BI 应用产品/服务条目。
-   [发布 Power BI 应用服务](./cpp-publish-offer.md)。 如何将新产品/服务提交到 AppSource 进行发布，以及如何更新现有产品/服务。


## <a name="publishing-steps"></a>发布步骤

下面是发布 Power BI 应用程序产品/服务的高级步骤：

![Power BI 应用提供发布步骤](media/publishing-steps.png)

下面是 Power BI 应用提供发布过程：

1. 在 Power BI 中创建模板应用程序。 此操作将生成包安装 URL，该 URL 表示产品/服务的主要技术资产。 同时，此时将测试包提升到预生产。 有关详细信息，请参阅[什么是 Power BI 模板应用？](https://docs.microsoft.com/power-bi/service-template-apps-overview)。 
2. 收集或创建产品/服务的营销材料，包括正式名称、说明、徽标等。 
3. 收集或创建产品/服务的法律和支持文档：*使用条款*、*隐私策略*、*支持策略*、用户帮助等。
4. 创建产品/服务：使用云合作伙伴门户可以配置产品/服务的详细信息，包括产品/服务说明、营销材料、法律信息、支持信息和资产规格。  完全指定产品/服务后，将其提交以供发布。
5. 监视云合作伙伴门户中的发布过程。  在此步骤中，AppSource 载入团队测试、验证和验证应用程序。 
6. 成功认证应用后，请在其测试环境中查看并发布该应用。 
7. Power BI 应用程序在 AppSource （它是 "上线"）上列出。
8. 在 Power BI 中，将预生产包提升到生产。 有关详细信息，请参阅[管理模板应用版本](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release)。


## <a name="next-steps"></a>后续步骤

创建 Power BI 应用产品/服务并将其发布到 AppSource 之前，必须满足将 Power BI 应用发布到 AppSource 的[要求](./cpp-prerequisites.md)。
