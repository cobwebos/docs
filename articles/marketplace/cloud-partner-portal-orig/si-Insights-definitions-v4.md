<a name="seller-insights-definitions"></a>Seller Insights 定义
=======================

下表提供了 Seller Insights 中使用的许多术语的定义。

|  **术语**                 |  **定义**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Azure 许可证类型**                               | 购买 Azure 的客户使用的许可协议类型。  也称为“渠道”。                                                  |
| **Azure 许可证类型：云解决方案提供商**      | 最终客户通过其云解决方案提供商（充当经销商）购买 Azure 和市场套餐。                 |
| **Azure 许可证类型： Enterprise**                   | 最终客户通过直接与 Microsoft 签署的企业协议购买 Azure 和市场套餐。                  |
| **Azure 许可证类型：企业到经销商**  | 最终客户 procures Azure 和你的 Marketplace 通过分销商提供便利，使其与 Microsoft 企业协议。     |
| **Azure 许可证类型：即用即付**                | 最终客户 procures Azure 和你的 Marketplace 通过 "即用即付" 协议，直接与 Microsoft 进行签名。                |
| **收费金额（CC）**                              | 向客户收取的费用，以*客户货币*（CC）计费。                                                                 |
| **收费金额（PC）**                               | 以*支出*（PC）为客户支付的金额。                                                                      |
| **收费日期**                                      | 计算客户费用的日期（通常在使用期之后）。                                             |
| **云实例名称**                              | 部署 VM 的 Microsoft 云。                                   |
| **Cloud Instance Name： Azure Global**                | 公共全局 Microsoft 云。                           |
| **云实例名称： Azure 政府版**        | 政府特定的 Microsoft 云，适用于以下政府之一：中国、德国或美国美国。                           |
| **集合状态**  | 对于特定费用，为计费和收集生命周期的最新状态。  例如，状态可能为 "正在收集"、"已收集" 或 "退款"。                  |
| **核心大小**  | 运行该产品/服务的虚拟机提供的虚拟核心数。               |
| **客户城市**  | 客户提供的城市名称。 这可能不同于客户的 Azure 订阅中的城市。  |
| **客户通信语言**        | 客户的首选通信语言。                                  |
| **客户公司名称**                  | 客户提供的公司名称。 这可能不同于客户的 Azure 订阅中的城市。                                  |
| **客户所在国家/地区**                       | 客户提供的国家/地区名称。  这可能不同于客户的 Azure 订阅中的国家/地区。                               |
| **客户货币（CC）**                 | 客户首选的定价和计费货币。                            |
| **客户电子邮件**                         | 最终客户提供的电子邮件地址。  这可能不同于客户的 Azure 订阅中的电子邮件地址。                   |
| **客户名字**                              | 客户提供的名字。  这可能不同于客户的 Azure 订阅中提供的名称。                    |
| **客户姓氏**                               | 客户提供的最后一个名称。  这可能不同于客户的 Azure 订阅中提供的名称。                    |
| **客户 ID**                               | 分配给客户的唯一标识符。  一个客户可以有零个或多个 Azure Marketplace 订阅。                    |
| **客户付款类型**                            | 客户使用的付款方式类型。  例如，这可以是卡、发票或电子检查。                             |
| **客户邮政编码**                             | 客户提供的邮政编码。  这可能与客户的 Azure 订阅中提供的邮政编码不同。                                                                 |
| **客户状态**                                   | 客户提供的省/市/自治区（address）。  此名称可能不同于客户的 Azure 订阅中提供的姓氏。                                                                 |
| **获取日期**                                    | 客户购买您发布的任何产品/服务的第一天。                 |
| **丢失日期**                                        | 客户取消上次购买的所有产品/服务的最后日期。                                                         |
| **估计的扩展费用（CC）**                   | 给定 SKU 的用量（以客户的货币表示）的用量的估计延伸费用。 由于舍入或截断错误，此值可能不精确。                            |
| **估计的扩展费用（PC）**                   | 计算针对给定 SKU 的使用量单位的使用量（以出版商的币种表示的基于外汇转换的给定 SKU 的使用量）。 由于舍入或截断错误，此值可能不精确。                  |
| **估计支出（PC）**                            | 针对给定 SKU 的使用量单位的估计付款量，基于对使用情况的日期（以出版商的货币计算）。 由于舍入或截断错误，此值可能不精确。                        |
| **估计价格（PC）**                             | 根据日期使用情况的外汇转换，计算给定 SKU 的使用单位的预计价格（以出版商的币种表示）。 由于舍入或截断错误，此值可能不精确。                                             |
| **是新客户**                                  | 通过此值，你可以了解它是新客户，还是首次购买一个或多个 Marketplace 产品/服务。  如果 "已获取日期" 是同一日历月份，则该值将为 "是"。  如果客户在报告的日历月之前购买了你的任何产品/服务，则该值将为 "No"。                                                       |
| **为预览版 SKU**                                  | 如果已将 SKU 标记为 "预览"，则该值将告诉你。 如果已相应地标记了 SKU，则该值将为 "是"，并且只有授权的 Azure 订阅才能部署和使用此映像。  如果 SKU 未被标识为 "预览"，则该值将为 "No"。                                          |
| **促销联系人选择加入**                   | 此值可让用户了解客户是否主动选择加入发布者提供的促销联系人。 目前我们不会向客户提供该选项，因此，在整个板块中都指示了“否”。 部署此功能后，我们会相应地做出更新。                                          |
| **Marketplace 许可证类型**                         | 市场套餐的计费方式。                             |
| **Marketplace 许可证类型：通过 Azure 计费**   | Microsoft 是此市场套餐的代理，并代你向客户计费。 （PAYG 信用卡或企业发票）       |
| **Marketplace 许可证类型：自带许可证** | VM 需要客户提供的许可证密钥才能进行部署。 Microsoft 不会向使用此市场套餐的客户计费。 |
| **Marketplace 许可证类型：免费**                   | 市场套餐配置为供所有用户免费使用。 Microsoft 不会向使用此市场套餐的客户计费。    |
| **Marketplace 许可证类型： Microsoft as 经销商**  | Microsoft 是此市场套餐的经销商。                               |
| **Marketplace 订阅公司名称**  | 客户在其 Azure 订阅中提供的公司名称。                           |
| **Marketplace 订阅 ID**            | 与客户用于购买 Marketplace 产品/服务的 Azure 订阅相关联的唯一标识符。  它以前是 Azure 订阅 GUID。  在某些情况下，如 SAAS 顺序，无需订阅。           |
| **Microsoft 费用（CC）**                               | Microsoft 收取的交易手续费，按客户的货币结算。                              |
| **产品/服务名称**                                | Marketplace 产品/服务的名称。                                                             |
| **产品/服务类型**                                | Microsoft 市场产品的类型。  例如，它可以是托管应用程序、虚拟机许可证或容器。                           |
| **订单取消日期**                         | 取消市场订单的日期。                                                       |
| **订单 ID**                                  | 适用于 Marketplace 服务的客户订单的唯一标识符。  基于虚拟机使用情况的产品/服务不与订单相关联。                 |
| **订单采购日期**                       | 创建 Marketplace 订单的日期。                                                        |
| **订单数量**                       | 与 Azure 订阅关联的同一 SKU 的订单数量。  对于 SAAS 订单，它表示与订单的此实例关联的座位数。                                            |
| **订单状态**                              | 上次刷新数据时市场订单的状态。  例如，可以激活或取消订单。             |
| **订单状态：活动**                           | 客户已购买订单，但尚未取消订单。   |
| **订单状态：已取消**                        | 客户以前购买了订单，并随后取消了订单。        |
| **支出（PC）**                             | 支付给你的金额，采用你喜欢的费用（PC）。                                       |
| **支出（PC）**                           | 付款时使用的货币。                                                                |
| **付出日期**                                    | 支付请求从 Microsoft 发送到所选金融机构的日期。                                                                         |
| **支出状态**                                  | 上次刷新数据时的支出状态。                                   |
| **付出状态：正在进行**               | 交易尚未做好付款的准备。 （有关详细信息，请参阅集合状态）                             |
| **支出状态：已支付**                          | 交易包含在以往付款计算中。 正值表示已付，负值是与逾期未付总金额相差的净额。 |
| **支出状态：即将发生的比率**                   | 交易已做好付款的准备，将包含在下一次应付款计算中。                      |
| **价格（CC）**                                   | 给定 SKU 的一个使用单位的价格（采用客户的货币）。                                                                  |
| **提供商电子邮件**                                 | Microsoft 与最终客户之间的关系所涉及的提供商的电子邮件地址。 如果客户是通过分销商的企业，则它将是分销商。  如果涉及到云解决方案提供商（CSP），则它将是 CSP。                                                                 |
| 提供者名称                                 | Microsoft 与最终客户之间的关系所涉及的提供者的名称。 如果客户是通过分销商的企业，则它将是分销商。  如果涉及到云解决方案提供商（CSP），则它将是 CSP。                                                                 |
| **资源 URI**                                  | 部署项目的部署路径。                                            |
| **SKU**                                           | 在发布过程中定义的 SKU 名称。 产品/服务可能有许多 Sku，但 SKU 只能与单个产品/服务相关联。              |
| **SKU 计费类型**                                 | SKU 的计费方法。                                                                                                             |
| **SKU 计费类型：自带许可证**         | VM 或服务需要客户提供的许可证密钥才能进行部署。 Microsoft 不会向客户收取这些 Marketplace 产品/服务的使用费用。   |
| **SKU 计费类型： Microsoft As 经销商**          | Microsoft 是此 SKU 的分销商。   |
| **SKU 计费类型：免费**                           | SKU 已配置为可供所有用户免费。 Microsoft 不会向客户收取此 SKU 的使用费用。                           |
| **SKU 计费类型：付费**                           | Microsoft 是此 SKU 的代理，会代表你向客户收费。 （PAYG 信用卡或企业发票）                   |
| **SKU 计费类型：试用版**                          | 客户处于套餐的试用期，如果不取消或删除订阅，套餐将转换为付费版。                                         |
| **税额（CC）**                                  | 以客户货币（CC）应用于客户帐单的税额。                                                               |
| **交易日期**                                 | 付款报告中的交易记录日期。                                                                           |
| **事务类型**                               | 正在报告的交易类型。  例如，这可能是收费、退款或支出调整。                                                                                        |
| **试用结束日期**                                 | 此订单的试用期将要结束或已结束的日期。                                                                           |
| **使用日期**                                       | 客户使用日期。                                                                                                             |
| **使用数量**                                       | SKU 的客户使用情况。 对于 VM 映像，使用记录表示该 VM 大小和 SKU 的报告期间的使用情况。                                               |
| **使用情况引用**                                       | 针对与付款报表中某个条目关联的给定 SKU，客户使用的一个或多个日期的标识符。                                               |
| **使用类型**                                       | 要度量的使用情况的说明。 （规范化使用情况或原始使用情况）                                                              |
| **使用类型：标准化用法**                | 根据使用时涉及的 VM 核心数规范化的使用小时数：[VM 核心数] x [原始使用小时数]。 指定为“SHAREDCORE”的 VM 使用 1/6（或 0.1666）作为 [VM 核心数] 的乘数。                                        |
| **使用类型：原始使用情况**                        | 市场套餐的使用小时数。  它以 "日历" 小时为单位显示。  虚拟机已运行的时间。                           |
| **使用单位**                                   | 所述使用情况的度量单位。 VM 始终以小时作为度量单位来度量。                               |
| **VM 大小**                                       | 表示与 Azure 套餐相符的虚拟机硬件大小。 示例包括 `Basic_A0`、`Standard_A11`、`Standard_D12` 和 `Standard_G4`。   |
|  |  |
