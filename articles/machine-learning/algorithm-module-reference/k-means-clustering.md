# <a name="k-means-clustering"></a>K 平均值聚类分析

本文介绍如何使用**K-means 聚类分析**Azure 机器学习工作室创建未经训练的 K 平均值聚类分析模型中的模块。 
 
K 平均值是一个简单的最广为人知*无人监督*学习算法，并可用于各种机器学习任务，例如[检测异常数据](https://msdn.microsoft.com/magazine/jj891054.aspx)、 聚类分析的文本文档和数据集之前使用其他分类或回归方法的分析。 若要创建聚类分析模型，将此模块添加到你的试验、 连接的数据集和设置参数，例如期望的要在创建群集，使用距离指标和等等的分类数。 
  
配置模块超参数后，连接到的未训练的模型[训练聚类分析模型](train-clustering-model.md)标签列的 K 平均值算法是一个非监督式的学习方法，因为是可选的。 

+ 如果你的数据包含一个标签，可以使用的标签值引导，群集的所选内容和优化模型。 

+ 如果你的数据没有标签，该算法将创建表示可能的类别，完全根据数据的群集。  
  

  
##  <a name="understanding-k-means-clustering"></a>了解 k 平均值聚类分析
 
一般情况下，聚类分析使用迭代技术在数据集中的用例包含类似特征的分类。 这些分组可用于研究数据、 标识数据中的异常并最终进行预测。 聚类分析模型还可以帮助您识别数据集不可能逻辑上得出通过浏览或简单的观察中的关系。 出于这些原因，聚类分析通常用于机器学习任务的早期阶段中浏览数据并发现意外的关联。  
  
 在配置使用 k 平均值方法对聚类分析模型时，必须指定目标数*k*指示的数*质心*想在模型中。 质心是代表每个分类的点。 K 平均值算法通过最小化平方群集中和将每个传入的数据点分配给其中一个群集。 
 
在处理定型数据，K 平均值算法开始一组初始的 randomnly 选择质心，它作为起点为每个群集，并应用 Lloyd 算法以迭代的方式优化中心位置。 K 平均值算法会停止构建和优化聚满足一个或多个这些条件时：  
  
-   质心稳定，这意味着不能再更改的各个点的群集分配和算法已收敛于一种解决方案。  
  
-   此算法已运行指定的迭代次数。  
  
 完成训练阶段之后, 您使用[将数据分配到群集](assign-data-to-clusters.md)模块以将新的用例分配给其中一个的 k 平均值算法发现的群集。 聚类分配执行通过计算新的用例的每个聚类质心之间的距离。 每个新的用例分配给具有最接近的质心的群集。  

## <a name="how-to-configure-k-means-clustering"></a>如何配置 K 平均值聚类分析
  
1.  添加**K-means 聚类分析**模块在试验。  
  
2.  指定要通过设置训练的模型的方式**创建训练器模式**选项。  
  
    -   **单一参数**:如果你知道你想要在聚类分析模型中使用的确切参数，您可以提供一组特定的值作为参数。  
  
   
  
3.  有关**数量的质心**，键入你想要开始使用的算法的分类数。  
  
     该模型不保证能够产生完全此数量的群集。 Algorithn 开始此数量的数据点，循环访问要查找的最佳配置。  
  
     
  
4.  属性**初始化**用来指定用于定义初始群集配置的算法。  
  
    -   **第一个 N**:一些初始数据点数目是从数据集选择，并且用作初始平均值。  
  
         也称为*Forgy 方法*。  
  
    -   **随机**:该算法在群集中随机放入一个数据点，然后计算初始平均值作为群集的形心随机分配点。  
  
         也称为*随机分区*方法。  
  
    -   **K-平均值 + +**:这是初始化群集的默认方法。  
  
         **K 平均值 + +** 算法提议，在 2007 年由 David Arthur 和 Sergei Vassilvitskii 以避免不佳聚类分析由标准 k 平均值算法。 **K 平均值 + +** 通过使用另一种方法选择初始聚类中心改进了标准 K 平均值。  
  
    
5.  有关**随机种子**，可以选择键入要用作群集初始化种子的值。 此值可以具有重大影响群集所选内容上。  
  
    
  
6.  有关**指标**，选择要用于衡量群集向量之间或新的数据点和随机选择的质心之间的距离的函数。 Azure 机器学习支持以下群集距离度量值：  
  
    -   **欧几里得**:欧几里得距离通常用于衡量群集散点图 k-means 聚类分析。 此指标是首选，因为它降至最低点和质心之间的平均距离。
  

  
7.  有关**迭代**，键入该算法应循环访问训练数据在最终确定质心的所选内容之前的时间量。  
  
     您可以调整此参数为平衡准确性与培训时间。  
  
8.  有关**分配标签模式**，选择一个选项，指定标签列，如果存在在数据集中，应如何处理。  
  
     由于 K 平均值聚类分析的无人监督的机器学习方法，标签是可选的。 但是，如果你的数据集已有标签列，可以使用这些值来指导所选内容的群集，或可以指定这些值被忽略。  
  
    -   **忽略标签列**:标签列中的值将被忽略，而不使用在构建模型。
  
    -   **填充缺失值**:标签列的值用作功能来帮助生成此分类。 如果任何行缺少一个标签，则这被处理使用其他功能。  
  
    -   **从靠近中心覆盖**:标签列的值替换为使用与当前质心最接近的点的标签预测的标签值。  

8.  选择的选项**规范化特征**，如果希望在定型之前的特征进行规范化。
  
     如果应用规范化，在定型之前, 的数据点被规范化为`[0,1]`MinMaxNormalizer 通过。

10. 训练该模型。  
  
    -   如果您设置**创建训练器模式**到**单个参数**，添加标记的数据集，并使用来训练模型[训练聚类分析模型](train-clustering-model.md)模块。

### <a name="tips-for-generating-the-best-clustering-model"></a>用于生成聚类分析模型的最佳提示  

它已知**种子设定**聚类分析过程中使用的过程，则会对模型产生重大影响。 种子设定意味着到 potental 质心点的初始位置。
 
例如，如果数据集包含许多离群值，并选择离群值的种子分类，没有其他数据点将适合于该群集和群集可以是单一实例： 即，具有只有一个点的群集。  
  
有多种方法来避免此问题：  
  
-   更改质心数并尝试多个种子值。  
  
-   创建多个模型，不同的度量值或循环访问的详细信息。  
  
  
一般情况下，使用聚类分析模型，就可能任何给定的配置将导致本地优化的一组分类。 换而言之，群集由模型返回的集适合仅当前的数据点，并不是可归纳为其他数据。 如果你使用不同的初始配置，则 K 平均值方法可能会发现不同，或许是高级配置。
